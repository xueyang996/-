# git 开发流程
1. 开发新需求时，先从master（或develop）分支拉取最新代码，以feature开头创建自己的新分支
2. 新功能开发完成，提交代码，发起merge请求（请求合并到master或develop分支）
3. merge之前，为了防止冲突，建议将master分支的最新代码先merge到自己的开发分支
4. merge之后，删除自己的分支
